# Telegram-бот с простыми рецептами

Бот на Python (`python-telegram-bot`) для подбора простых и вкусных рецептов, меню и списка покупок.

## Что умеет бот

- Подбор рецептов на завтрак, обед, ужин и быстрых рецептов
- Генерация меню на день и на неделю
- Перегенерация меню на день/неделю кнопкой `🔄 Перегенерировать меню`
- Формирование списка выбранных рецептов
- Генерация списка покупок только по рецептам, которые пользователь сам добавил кнопкой `Добавить в покупки 🛒`
- В списке покупок можно отмечать купленные позиции и добавлять свои
- Подбор блюд по имеющимся продуктам (`/изчего`)
- Поиск рецептов по названию блюда
- Фильтрация блюд: низкокалорийные, белковые, дешевые
- Избранное рецептов (`/избранное`)
- Учет предпочтений пользователя через исключение ингредиентов (`/предпочтения`)
- Загрузка рецептов из файла `recipes_110_final.json`
- Вывод КБЖУ для каждого рецепта
- Кнопка `🔄 Обновить список рецептов` в подборках
- Хранение данных пользователей в `data.json`

## Быстрый запуск

1. Создайте бота в Telegram через [@BotFather](https://t.me/BotFather) и получите токен.
2. Установите Python 3.10+.
3. В папке проекта:

```bash
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
cp .env.example .env
```

4. Откройте `.env` и вставьте токен:

```env
BOT_TOKEN=ваш_токен_из_BotFather
```

5. Запустите бота:

```bash
python bot.py
```

## Где хранятся данные

### 1) Рецепты

Файл `recipes_110_final.json` используется как основной источник рецептов.

Бот ожидает в рецепте поля:

- `name` — название блюда
- `meal` — категория: `breakfast`, `lunch`, `dinner`
- `quick` — быстрый рецепт (`true/false`)
- `time` — время приготовления в минутах
- `ingredients` — список ингредиентов
- `steps` — шаги приготовления
- `servings` — порции (необязательно)
- `nutrition` — КБЖУ на порцию в формате:
  - `kcal`
  - `protein`
  - `fat`
  - `carbs`

Если JSON поврежден, бот пытается восстановить рецепты из корректных фрагментов и продолжить работу.

### 2) Данные пользователей

Файл `data.json` создается автоматически при первом взаимодействии пользователя с ботом и содержит:

- избранные рецепты
- выбранные (просмотренные) рецепты
- отдельный список рецептов для покупок
- предпочтения пользователя (исключенные ингредиенты)

## Обработка ошибок

В боте есть глобальный обработчик ошибок:

- ошибки логируются в консоль
- пользователю отправляется дружелюбное сообщение об ошибке
- при пустом или недоступном `recipes_110_final.json` бот завершает запуск с понятной ошибкой
